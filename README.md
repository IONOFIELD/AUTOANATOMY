# AUTOANATOMY

A tool to "chunk" the sections of vehicles and their functions like an anatomy
learning tool designed around automotive technology.

First vehicle: **Mercedes-Benz W221 S65 AMG** — peel it apart layer by layer
(body panels → chassis/suspension → wiring harness → M275 V12 biturbo engine →
engine internals) and tap any part for its name, system, function and
description, Complete Anatomy style.

Built with **Vite + React + TypeScript + react-three-fiber + drei**. No backend.

The vehicle view uses a real W221 model (see **Credits**) merged at runtime with
generated wheels and wiring harness; the engine view is fully procedural so the
internals (crank, pistons, cams, timing chain) can be peeled like an anatomy app.

## Quick start

```bash
npm install
npm run dev        # http://localhost:5175
```

Optional:

```bash
npm run build            # typecheck + production build to dist/
npm run generate:models  # regenerate the procedural GLBs (engine, extras, fallback vehicle)
npm run prepare:w221     # re-bake the real vehicle GLB from assets-src/w221-original.glb
```

## How it works

### The peel model (skin → muscle → bone)

Layers are ordered outermost (1) to innermost (5) in
`public/config/models.json`. The app keeps a single **peel depth**: selecting a
layer hides that layer *and everything outboard of it*, revealing what's
beneath. Selecting an already-peeled layer restores down to it. "Show all"
resets to depth 0.

| # | Layer id    | Contents |
|---|-------------|----------|
| 1 | `body`      | Body panels, glass, bumpers |
| 2 | `chassis`   | Frame/floorpan, subframes, suspension, wheels |
| 3 | `harness`   | Wiring harness (simplified), battery, SAM/fuse boxes |
| 4 | `engine`    | M275 externals: block, banks, turbos, intake, ancillaries |
| 5 | `internals` | Crankshaft, pistons, rods, camshafts, valvetrain |

### Linked models & engine focus

There are **two separate GLB models**, not one mesh:

- `vehicle.glb` — exterior shell, chassis, harness, plus a low-detail engine
  *proxy* so the bay isn't empty at layer depth 3.
- `engine.glb` — the standalone detailed M275 used for layers 4–5.

The **Focus** buttons in the top bar swap the scene/camera between them.
Focus also swaps automatically: peeling into layer 4 (engine externals hidden →
internals visible) jumps to the engine model, because only it has internals;
restoring back out to layers 1–3 returns to the vehicle. Which model "owns"
which layers is configured per model via `focusLayerIds` — no code involved.

### Controls

- **Orbit**: drag (1 finger) · **Pan**: right-drag / 2-finger drag ·
  **Zoom**: scroll / pinch
- **Tap/click a part**: opens the info panel (name, system, function,
  description). Tap empty space to close.
- Layout is responsive: sidebar layer stack on desktop, bottom chip bar +
  bottom-sheet info panel on phones.

## Adding a real / new model

Mesh names are **never hardcoded** in app code — everything is driven by
`public/config/models.json`. To drop in a real scanned/CAD model:

1. Export it as a `.glb` with **stable, meaningful mesh names** (ideally
   prefixed per system, e.g. `Body_Hood`, `Susp_Strut_FL`). Put it in
   `public/models/`.
2. Register it in `public/config/models.json` under `"models"`:

   ```jsonc
   {
     "id": "vehicle-real",
     "name": "W221 (scan)",
     "url": "models/w221-scan.glb",
     "focusLayerIds": ["body", "chassis", "harness", "engine"],
     "camera": { "position": [5.4, 2.8, 6.6], "target": [0, 0.7, 0] },
     "meshLayerMap": { /* see next section */ }
   }
   ```

   - `focusLayerIds`: the layers this model can display. The app uses this to
     decide when to auto-swap focus between models.
   - `camera`: initial camera position/orbit target for this model (metres).
   - Use `"files": ["a.glb", "b.glb"]` instead of `url` to combine several
     GLBs into one model — that's how the real car is merged with the
     generated wheels/harness.
   - `"glassMeshes": ["pattern*"]` forces matching meshes to render as glass —
     imported models often ship glass as opaque materials.
3. Reload. Meshes not covered by `meshLayerMap` stay always-visible and log a
   `console.warn` with their exact name — open devtools to harvest the names
   you still need to map. Patterns match each mesh's own name **and its
   ancestor group names**, so a GLB whose meshes are named `Object_53` under a
   group `sw221_hood_fl` maps cleanly via `"sw221_hood_*"`. The same ancestor
   walk names parts in the info panel and in `parts.json`.

To add a new **layer** (e.g. "Interior"), add it to `"layers"` with the right
`order` and reference its id from a model's `meshLayerMap` — the peel stack UI
is generated from this list.

> Tip: if your GLB shares one material across many meshes, the app clones
> materials automatically on load so per-part highlighting works.

## Importing downloaded models (Sketchfab, Assetto Corsa, …)

The fastest realism upgrade is a downloaded model. Two routes:

**Sketchfab (recommended — legal & already glTF).** Search for downloadable
models ("Download 3D Model" button, free account required); Sketchfab converts
every downloadable model to glTF/GLB for you. Check the license on the page:
CC-BY is fine — keep the author credit (add it to this README). Candidates
found for this project:

- Mercedes S-Class W221 by Black Snow — CC Attribution
  (sketchfab.com/3d-models/mersedes-benz-s-class-w221-6b1c0ad55406406a8cf8177dc0a4f2b9)
- "(FREE) V12 – 12 Cylinder V-Engine" by Exonl, "[Free] RWD V12 Engine" by
  Dev365TH, "V12 Twin Turbo Engine" by KeonCox — check each page's license.

**Assetto Corsa mods (last resort).** W221 S65 mods exist (e.g. the
"W221 S65 PRESIDENT" on assettoworld.com), **but**: (1) they ship as `.kn5`,
a proprietary format — you must convert on your PC via Content Manager
(showroom → export FBX) or the Blender kn5 importer add-on, then export GLB
from Blender; (2) most street-car mods are unauthorized ports from other
games, so treat them as personal-study material only and **do not commit
them to this repository**.

**After downloading either way:**

1. In Blender (free): open the model, delete interior/junk you don't need,
   and check the Outliner — you want *separate objects* for hood, doors,
   wheels, etc. Split (`P` → By Loose Parts) and rename if it's one blob.
   Keep it under ~2–3 M triangles for phones (use the Decimate modifier).
2. Export as GLB into `public/models/`, register it in
   `public/config/models.json`, and map its mesh names to layers (next
   section). Unmapped meshes stay visible and log their names in the browser
   console — the quickest mapping workflow is: load, open devtools, copy the
   warned names into `meshLayerMap`.
3. Local-only assets: if you must keep an asset out of git, add its filename
   to `.gitignore` and note where to get it here in the README.

## Mapping meshes to layers (`meshLayerMap`)

Each model has its own map of mesh-name pattern → layer id:

```json
"meshLayerMap": {
  "Body_*": "body",
  "Chassis_*": "chassis",
  "Susp_*": "chassis",
  "Harness_*": "harness",
  "EngineProxy_*": "engine"
}
```

Pattern rules (shared with `parts.json`):

- An exact mesh name (`"Body_Hood"`) always wins.
- A pattern ending in `*` matches any mesh with that prefix
  (`"Susp_*"` → `Susp_Wheel_FL`, `Susp_ABCStrut_RR`, …).
- If several prefixes match, the **longest** one wins, so you can map
  `"Eng_*": "engine"` broadly and override one mesh with
  `"Eng_SpecialThing": "internals"`.

## Editing `parts.json`

`public/data/parts.json` is the entire content database — add or edit entries
without touching code. Keys are mesh names (or `prefix_*` patterns), values:

```json
"Eng_Turbo_*": {
  "name": "Turbocharger",
  "system": "Engine — forced induction",
  "function": "Uses exhaust energy to compress intake air, one turbo per bank.",
  "description": "Two water-cooled turbochargers give the S65 tune ~1.5 bar of boost…"
}
```

- `"Int_Piston_*"` covers all twelve pistons with one entry; add
  `"Int_Piston_1"` alongside it to give cylinder 1 its own text.
- Clicking a part with no entry shows a friendly fallback naming the exact
  mesh key to add — so you can build the database by clicking around.
- It ships pre-seeded with ~30 W221/M275 components as examples.

## Placeholder models

The two GLBs are generated procedurally (`npm run generate:models`) at roughly
PS2-era fidelity — 80 meshes for the vehicle (sedan silhouette with wheel-arch
cutouts, pillars, glass, tires/rims/rotors/calipers, coil-spring ABC struts,
driveshaft, exhaust with mufflers) and 60 for the engine (crankshaft with real
throws, phased pistons and rods, cam lobes, valves, timing chain, turbo
scrolls, serpentine belt). They exist so the app runs before real assets are
sourced; replacing them is just "drop in GLB + edit `models.json`" as
described above.
